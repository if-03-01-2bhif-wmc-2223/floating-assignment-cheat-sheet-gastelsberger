Web Client: The web client is also known as browser. It gets information from the web server (via the internet). The client serves as application software (program designed to carry out a specific task) for accessing the world wide web. The client does things such as rendering, fancy GUI stuff and it requests a new page when required.


Process: A URL is entered to the browser, which retrieves information from the browser using the appropriate protocol. Communication between the browser and the web server is encrypted for security/privacy reasons. Once the pages were retrieved, the rendering engine displays it on the users device. When a hyperlink is clicked, a new resource will be opened. Most browsers have an internal cache to reduce loading times.

Web Server:
A webserver is a computer software with underlying hardware that accepts requests in HTTP or HTTPS. It can contain pre-existing files (static content) or the content can be generated at the time of the request (dynamic content). More information to be found below.

The web server houses HTML content.

The server side static houses files, which can be retrieved at some point. There are no scripts whatsoever in that type of webpage.

Server side dynamics exist too, applying to sites that are powered by scripts to make them dynamic (kind of obvious).

Then you also have database driven webpages, which are scripted and have a database behind it. The scripting makes a query and receives data in return.

The internet was invented by Sir Tim Berners-Lee at the CERN (European Organization for Nuclear Research) at around 1989. He had the first World Wide Web Browser and a HTTP server ready by the end of 1990. He also founded the WWC Consortium (W3C), which created XML (Extendible Markup Language).

W3C recommended replacing HTML with XHTML, which failed and instead HTML5 was developed by global players.

'URL' is the abbreviation for Uniform Resource Locator, colloquially known as web address. The purposse of a URL is to identify and locate resources, such as websites, using a protocol and the resource in a network. The syntax is simple:

PROTOCOL :// PATH ? #
The paths are seperated with a slash, the '?' is a query and # is the fragment.

The correct doctype declaration is "!DOCTYPE html" (don't forget the greater/smaller characters).

HTML elements have a starting tag, (name) and an ending tag. Void elements do not have end tags and can't house any content inside it. These elements have attributes and can't be nested.

The id atribute is a unique ID for a HTML element and must be unique within the HTML document. This attribute is used to point to a specific style declaration in a style sheet. JavaScript also uses this to access and manipulate the element with the specific id.

CSS

CSS stands for Cascading Style Sheets.
Every css rule has a selector, a value, a property and a declaration.

h1 // SELECTOR
{
color (property): blue (value);
}

WARNING!
The div element is only a placeholder in the following cases.

Direct child: Selects all elements that are children of a specified element. (Selects all p's that are children of a div element)
div > p {color: blue;}

Descendant: Selects all elements that are children of a specific element. (Selects all p's inside div elements)
div p {color: blue;}

Adjacent sibling: Selects the first p element of a div element.
div + p {color: blue;}

Sibling: Selects all elements that are siblings of a specific element. (Let's say you have 4 paragraphs. Two within a div tag, two not in a div tag. The two statements not within the div tag will have the color blue.)
div ~ p {color: blue;}

Colors in a hexadecimal notation:
(opening) h1 style="color:#ff3813;" (closing) h1

Declaration for background image:
body{
background-image: url("yes.png");
}

Text alignation (left, right, center, justify):
h1{
text-align: left;
}

Text decoration (overline, underline, line-through):
h1{
text-decoration-line: overline;
}

Text transformation (uppercase, lowercase, capitalize):
h1{
text-transform: uppercase;
}

Font family declaration:

Lists and Tables

List Types (and their styling) Explained

You differentiate between ordered lists (tag: ol) and unordered lists (tag: ul). To put an item in a list, you need to add a list item (li).

Description lists (tag: dl) are lists with a description of each term. The term is defined using the dt tag, term explanations are added using the dd tag.

Styling Lists

A list looks better with bulletpoints or squares. To add them, specify what you would like to add.

You should style lists using the list-style tag.

Examples:
Circle
- displays a circle.
Square
- displays a square.
Upper Roman
- displays (capital) roman numerals.
Lower Alpha
- displays numbers that we know (and love).

Positioning lists can be done using list-style-position: inside OR outside.

Default settings can be removed. You'd have to set the margin (space) and padding (also the space) to zero and change the list-style-type to zero.

HTML Tables (and their styling) explained

A table is defined using the table tag. Table rows are defined using the tr tag. Table cells are defined using the td tag. Sometimes, if you have longer lists, you want to have a header. Headers can be defined using the th tag.

Useful Tags:
Caption
- gives the table a title.
Colgroup
- selects a group of one or more columns for formatting.
Col
- specifies column properties for each column within a colgroup element.
thead, tbody, tfoot
- groups the content of that type in a table.

Borders

There are quite a few different border styles. Use these styles by typing "border-style:"

  • dotted
  • dashed
  • solid
  • double
  • groove (3D border, effect depends on border-color value)
  • ridge (3D)
  • inset (3D)
  • outset (3D)
  • none
  • hidden

The border width can be specified with the border-width property. (top, right, bottom, left if you want to declare the width of everything in one statement)

The border color can be set with the border-color property.

You can create individual borders. e.g.: border-(SIDE (left/right/top/bottom))-style: dotted

Round borders can be declared using the border-radius property (usually 5px).

Graphic Formats

jp(e)g

This format (Joint Photographic (Expert) Group) is pretty much the most important format for pictures and other elements that require more than 256 colors. Data about the brightness and colors are stored seperately. Support is provided for:

  • CMYK
  • RGB
  • grayscale color modes

Graphical content: picture

Compression: supported

Motion picture support: supported with MJPEG

gif

The grapics interchange format is an incredibly versatile format for the web with the ability to:

  • display pictures in a high quality
  • compromise pictures extraordinarily well
  • also be animated

Graphical content: picture (can also be animated)

Compression: looseless

Motion picture support: supported

png

Portable Network Graphics is an alternative to the GIF format. The following is supported:

  • greyscale color modes
  • RGB
  • ONLY if alpha-channels are there

Graphical content: picture

Compression: looseless

Motion picture support: supported with APNG

ico

ICO is primarily used for icons.

Graphical content: pictures/icons

Compression: looseless

Motion picture support: no motion support

svg

SVG are vector graphics and they're not dependant on pixles. That means, that the image will always look decent.

Graphical content: pictures

Compression: looseless

Motion picture support: supported

mpg/mp4

This format is used for videos and for sounds.

Graphical content: videos

Compression: not supported

Motion picture support: supported
